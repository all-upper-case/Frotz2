# Frotz2

Frotz2 is a Flask-based interactive fiction engine with a retro terminal UI and an LLM-assisted world model. It combines deterministic text-adventure mechanics with generated narration, rooms, and state updates driven by a lore file.

The project is designed for personal, local-first experimentation. Runtime saves, debug logs, and lore variants are part of the working process, while the core app lives in a small Python/Flask codebase.

## What It Does

- Serves a terminal-style browser UI with command input, markdown-rendered responses, and a side HUD.
- Maintains persistent world state for rooms, exits, items, inventory, worn objects, player description, and narrative history.
- Handles core commands deterministically, including `look`, `inventory`, movement, and `examine`.
- Uses an LLM backend for world genesis, unexplored-room generation, free-form player actions, and state harmonization.
- Provides local workflow helpers for debugging, model switching, and hands-on world-state editing.

## Project Layout

| Path | Purpose |
| --- | --- |
| `main.py` | Flask app, gameplay routes, command routing, model routes, and debug endpoints. |
| `world_manager.py` | Save/load logic, room and item state, movement, schema repair, and world mutation helpers. |
| `llm_interface.py` | Venice-compatible chat API calls, model listing, usage tracking, and debug logging. |
| `system_prompts.py` | Prompt templates for genesis, room generation, turn handling, and item-description fixes. |
| `templates/index.html` | Browser UI shell. |
| `static/script.js` | Frontend command flow, HUD updates, settings, debug modal, and world-state editor. |
| `static/style.css` | Retro CRT and clean dark theme styling. |
| `devtools.py` | Optional local/Replit helper routes for phone-based fetch and patch workflows. |
| `lore.txt` | Active world seed loaded by the engine. |
| `LoreBooks/` | Alternate lore seeds. |
| `saves/` | Runtime save files generated by play sessions. |
| `debug_logs/`, `debug_log.txt`, `last_turn_debug.txt` | Runtime diagnostics and LLM payload history. |
| `backups/` | Historical notes and archived save/doc artifacts. |

## Setup

Requirements:

- Python 3.11
- Poetry
- A Venice API key, or a compatible key exposed through the existing fallback environment variable

Install dependencies:

```bash
poetry install
```

Run locally:

```bash
poetry run python main.py
```

The app listens on `0.0.0.0:5000` by default. Replit runs the same Flask app through Gunicorn using the command in `.replit`.

## Environment Variables

| Variable | Required | Notes |
| --- | --- | --- |
| `VENICE_API_KEY` | Yes for AI features | Primary API key used by `llm_interface.py`. |
| `VENICE_MODEL` | No | Overrides the default model. |
| `MISTRAL_API_KEY` | No | Legacy fallback still accepted by the current code. |
| `DEVTOOLS_TOKEN` | Recommended if devtools are exposed | Required by `devtools.py` only when set. |

## Gameplay Loop

1. The browser loads `/get_state`.
2. If no save exists, `/reset` asks the LLM to generate the initial world from `lore.txt`.
3. Deterministic commands are handled directly by `main.py` and `world_manager.py`.
4. Free-form actions are passed to the LLM with compact world context.
5. The LLM returns JSON containing narration and state updates.
6. `WorldManager` applies those updates, saves the world, and returns a fresh HUD state.
